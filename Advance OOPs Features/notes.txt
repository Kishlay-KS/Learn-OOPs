1.) Abstraction : It is the process of hiding the implementation details and showing only the necessary features or interface to the user.
                  Focus on what an object does rather than how it does.

                  - It is acheived through abstract classes and interfaces : making a blueprint of how an object will behave without worying about the implementation details.
* Abstract Methods - Methods declared in abstract classes or interfaces and have no body. The implementation must be provided by the subclass or the class that implements the interface.
* Concrete Methods - These are methods with complete implementation in an abstract class. Subclasses inherit these methods, but they are also allowed to override them if necessary.

Note : Abstraction also provides security in a way that the user interacts with the system at a higher level, without the risk of them modifying or interacting with internal mechanisms directly.

e.g. abstract class Animal {
    void eat() {}

    abstract void sound() {}
}

    class Dog extends Animal {
        @Override
        void sound() {
            //
        }
    }

    class Cat extends Animal {
        @Override
        void sound() {
            //
        }
    }

eat() -> concrete method
sound() - abstract method

e.g. Interfaces - All methods in an interface are implicitly abstract, and it is used to represent a contract that the implementing classes must fulfill.

interface Animal {
    void sound();
    void eat();
}

class Dog implements Animal {
    @Override
    public void sound() {

    }

    @override
    public void eat() {

    }
}


2.) Static and Default Methods

    Static methods belong to class rather than its instance. It can only access the static members of the class and not any instance variables or instance methods.
    e.g. 
        class Example {
            static void staticMethod() {
                System.out.println("This is a static method.");
            }
        }

        class Main {
            public static void main(String[] args) {
                Example.staticMethod();  // Accessing static method directly using the class name
            }
        }
    
    Default Methods were introduced in Java 8. With the introduction of default methods, interfaces can now provide default implementations for methods.
    e.g. 
        interface Example {
            default void defaultMethod() {
                System.out.println("This is a default method.");
            }
        }

        class Demo implements Example {
            public static void main(String[] args) {
                Example obj = new Demo();
                obj.defaultMethod();  // Accessing default method
            }
        }

    * Allow users to add new methods to an interface without affecting existing classes that implement the interface.
    * If a class implements an interface with a default method, it can override the method if needed.
    * This solves backward compatibility and multiple interfaces implementing the same method problems.

    * Abstract Class Constructor: An abstract class can have constructors, but you cannot create an instance of the abstract class directly. The constructor is only called when a subclass object is created.
    * Subclass Constructor: When a subclass is instantiated, its constructor can call the constructor of the abstract class using the super() keyword.
    * An abstract class can extend an abstract class and may or may not implement is abstract methods.




INTERFACES - defines a contract that a class which implements the interfaces must provide the implementations.

e.g. interface CarInterface {
    public void start();
    public void numberOfGears();
}
    class ManualCar implements CarInterface {
        * Need to implement every method. i.e. 100% abstraction.
        * After Java 8, default methods got introduced, which does not requires to have 100% abstraction. These methods can be overridden as well.
        * We can create static objects inside an interface as well
    }

* Interfaces cannot have instance variables, any variable declared inside it is static final variable and does not require an object to call it.
e.g. interface CarInterface{
    static final int cnt = 0;
}

* Interfaces cannot have constructor as there are no instance variables.
* A class can implement multiple interfaces, but it will have to implement all the methods.
* Interface inheritance is possible, but again it will have to implement the methods of all the contracts.
e.g. interface Animal {
        void eat();
    }

    interface Mammal extends Animal {
        void walk();
    }

    class Human implements Mammal {
        @Override
        public void eat() {
            System.out.println("Human eats food.");
        }

        @Override
        public void walk() {
            System.out.println("Human walks on two legs.");
        }
    }

STATIC - keyword that can be applied to a variable, method or block.

* Static variables - belong to the class and not the object.
                   - can be directly accessed from the class name.
* Static methods - belong to the class and can be accessed using the class name itself.
                 - only static variables can be accessed via the static methods.
* Static blocks - used to initialize the values at the object creation, not widely used.
                - static {
                    val = 10;
                }

* Static things can only trigger static things, to access an instance variable or method, an object is required.
* Non-Static method can trigger a static method as these are inside the class.


INNER CLASSES - Any class that is defined another class.

* Types : 1.) Static Nested Class - static keyword is required to define.
                                  - can only access the static members of the outer class.
                                  - no instance of outer class is required.
                                  - e.g. OuterClass.InnerClass em = new OuterClass.InnerClass();
          * We cannot call non-static methods inside the static class through the class name, we will need and instance.
          2.) Non-Static Nested Class - tied to an instance of the outer class.
                                      - access to all the attributes and methods of the outer class is given.
                                      e.g. OuterClass.InnerClass ic = new OuterClass().new InnerClass();
                                      - as an outer class object is created, it will have access to all the outer class instance variables.
          3.) Local Inner Class - class defined inside method/block of the outer class.
                                - access of all outer but local variables from an inner class must be final or effectively final(not touched after initialized).
          4.) Anonymous inner class - class without name.
                                    - one time use.
                                    - used to implement contracts.
              e.g. interface Sample {
                public void execute();
              }

              Sample s = new Sample() {
                public void execute() {
                    //
                }
              }
              s.execute();

              ** Sample can be abstract also.

              e.g. ArrayList<Integer> list = new ArrayList<>() {
                @Override
                public boolean add(Integer val) {
                    //
                }
              }
              - this changes the default behaviour of the add method of the parent class. As we have overridden this with an anonymous inner class.
